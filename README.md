# Smart Attendance System

An AI-powered attendance management system that uses facial recognition to automatically mark student attendance from group photos. Built with a modern tech stack featuring React, Node.js, Flask, and DeepFace.

## 🏗️ Project Architecture

```
smart-attendance-system/
├── frontend-final/     # React + Vite frontend (Mantine UI)
├── backend/            # Node.js + Express REST API
└── ai-service/         # Python Flask + DeepFace AI service
```

```
│                 │     │                 │     │                 │

                        └─────────────────┘
```

## 📋 Features

### For Teachers
- **Create Classes**: Set up classes with unique enrollment codes
- **Enroll Students**: Add students with their photos for face recognition
- **Mark Attendance**: Upload group photos to automatically detect and mark present students
- **View Reports**: Track attendance history and generate reports

### For Students
- **View Attendance**: Check personal attendance records across enrolled classes
- **Dashboard**: See attendance statistics and class information

## 📊 Database Models

### Teacher
- Email, password, name
- Handles authentication and class management

### Student  
- Name, email, face embedding (vector)
- Linked to enrolled classes

### Class
- Name, enrollment code, teacher reference
- Contains list of enrolled students

### Attendance
- Class, date, list of present students
- Tracks daily attendance records

## 🔄 Application Flow

### 1. Teacher Registration & Login
```
Teacher signs up → Password hashed → JWT token issued → Access granted
```

### 2. Class Creation
```
Teacher creates class → Unique enrollment code generated → Class stored in DB
```

### 3. Student Enrollment
```
Teacher adds student → Photo uploaded to Cloudinary → 
Face embedding generated by AI service → Student stored with embedding
```

### 4. Attendance Marking (AI-Powered)
```
Teacher uploads group photo → Photo sent to Cloudinary →
AI service extracts all faces → Compares with enrolled student embeddings →
Matching students marked present → Attendance record saved
```

### 5. Student Access
```
Student logs in → Views enrolled classes → Checks attendance history
```

## 📁 API Endpoints

### Authentication (`/api/auth`)
- `POST /register` - Register new teacher
- `POST /login` - Login teacher/student
- `GET /me` - Get current user

### Classes (`/api/classes`)
- `GET /` - Get all classes for teacher
- `POST /` - Create new class
- `GET /:id` - Get class details
- `DELETE /:id` - Delete class

### Students (`/api/students`)
- `POST /` - Enroll new student
- `GET /class/:classId` - Get students in class
- `DELETE /:id` - Remove student

### Attendance (`/api/attendance`)
- `POST /mark` - Mark attendance with group photo
- `GET /class/:classId` - Get attendance records
- `GET /student/:studentId` - Get student's attendance

### Reports (`/api/reports`)
- `GET /class/:classId` - Generate attendance report
